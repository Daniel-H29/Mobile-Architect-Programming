CS 360 – Mobile App Development
Module Eight Journal Portfolio Submission

This repository contains my completed mobile application developed during Project Three. The submission includes the finalized source code and the refined user interface design originally created in Project Two. Together, these components demonstrate my ability to design, implement, and test a functional mobile application using user-centered design principles and mobile development best practices.

The primary goal of this project was to design and build a mobile application that solves a specific user need while maintaining usability, clarity, and efficiency. The application was developed to provide users with a streamlined experience that reduces friction when completing core tasks. Throughout the project, I focused on ensuring that the app was intuitive, responsive, and logically structured so users could easily navigate and accomplish their objectives. By aligning technical implementation with user expectations, the final product reflects both functional reliability and thoughtful design.

To support user needs, the application includes multiple screens organized in a logical and accessible navigation flow. Each screen was intentionally designed to serve a clear purpose, whether providing information, accepting user input, or guiding users through key tasks. The interface emphasizes simplicity, consistent layout, readable text, and clear interactive elements. I prioritized visual hierarchy and accessibility to reduce cognitive overload and make the experience as intuitive as possible. Designing the UI before implementing the code allowed me to ensure that development decisions directly supported the intended user experience, resulting in a cohesive and user-centered final product.

My development process followed a structured and incremental approach. I began by reviewing the UI designs and mapping each component to its required functionality. From there, I implemented features step by step, testing each component before moving forward. This approach allowed me to identify issues early and maintain a clean, organized codebase. I used descriptive naming conventions, maintained consistent formatting, and added comments where necessary to improve readability and maintainability. Breaking the project into manageable pieces helped me stay organized and ensured that each part of the application functioned correctly before integration.

Testing played a critical role in the development process. I conducted manual testing of all features, verified navigation between screens, and validated user inputs to ensure proper error handling. This iterative testing process allowed me to detect and correct logical errors and usability concerns before finalizing the app. Testing is essential in mobile development because it ensures reliability, improves user experience, and prevents unexpected behavior after deployment. Through testing, I gained a deeper understanding of how small issues can impact overall functionality and user satisfaction.

During development, I encountered challenges related to integrating user interface components with backend logic while maintaining simplicity and efficiency. In certain cases, I had to refactor code and rethink workflows to ensure that user actions triggered the appropriate functionality without creating unnecessary complexity. Overcoming these challenges required adaptability and problem-solving, particularly when balancing design clarity with technical implementation. By simplifying logic and restructuring portions of the code, I was able to maintain both usability and clean architecture.

Overall, this project demonstrates my ability to take a mobile application from initial planning and design through development, testing, and finalization. It highlights my understanding of user-centered design principles, structured coding practices, and iterative testing strategies.
